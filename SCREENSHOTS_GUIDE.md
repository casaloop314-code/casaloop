## How to Take Perfect Screenshots

### Using Chrome DevTools (Recommended)

1. **Open CasaLoop in Browser**
   ```
   http://localhost:3000 (development)
   https://your-app.vercel.app (production)
   ```

2. **Set Mobile Dimensions**
   - Press `F12` to open DevTools
   - Click "Toggle Device Toolbar" (Ctrl+Shift+M)
   - Select "iPhone 14 Pro" or custom: `390 x 844`
   - Set zoom to 100%

3. **Capture Screenshot**
   - Method 1: Use built-in Chrome screenshot
     - Open DevTools > Click 3 dots menu > "Capture screenshot"
   - Method 2: Use browser extension
     - Install "GoFullPage" or "Fireshot"
   - Method 3: System screenshot
     - Windows: `Win + Shift + S`
     - Mac: `Cmd + Shift + 4`

4. **Resize to Required Dimensions**
   - Use online tool: https://www.resizepixel.com
   - Or Photoshop/GIMP
   - Target: 1080 x 1920px

## Screenshot Enhancement Tips

### Before Taking Screenshots:

1. **Populate with Real Data**
   - Add 5-10 property listings
   - Create 3-5 service providers
   - Send test messages
   - Generate notifications

2. **Use Attractive Images**
   - Use high-quality property photos
   - Ensure consistent image quality
   - No placeholder images

3. **Clean UI State**
   - Clear any errors
   - Hide debug console
   - Remove test data labels
   - Ensure loading states are complete

4. **Add Sample Activity**
   - Set streak to 5+ days
   - Add 50+ CASA points
   - Show notifications (3-5 unread)
   - Display view counts on properties

### Image Editing (Optional but Recommended):

1. **Add Frames** - Use Figma or MockUPhone to add device frames
2. **Add Captions** - Overlay text highlighting features
3. **Use Gradient Backgrounds** - Make screenshots pop
4. **Show Hands Holding Device** - More engaging visuals

**Tools:**
- Device Frames: https://mockuphone.com
- Image Editing: Figma (free), Canva
- Screenshot Design: https://previewed.app

## Need Help?

If screenshots don't look right:
1. Ensure app is in production mode (not localhost if possible)
2. Clear browser cache and reload
3. Use incognito/private browsing mode
4. Check that all images are loading
5. Verify Firebase data is properly populated

**Ready to Submit!** Once you have 5-8 high-quality screenshots, head to Pi Developer Portal and upload them in the App Gallery section.
